#jsp01

<<jsp>>
: web 서버가 클라이언트에게 응답하기 위해 필요한 문서
	(주로 HTML 문서 만들어주는 언어)
	
	WEB Server가 가진 엔진이 실행할 수 있는 언어는 따로 있다.
		tomcat - java
	이것을 우리는 Server Side Page(SSP)라고 부른다.
	
	종류 
		1. ASP(Active Server Page) : Visual Basic
		2. ASP.NET : C#
		3. PHP : CGI
		4. JSP : Java
		
	서버 실행해서 클라이언트에게 응답하는 응답 문서를 만들기 위한 언어 = jsp
	
	서버가 실행할 수 있는 프로그램 만드는 것
	그 결과 파일로 만들어지며
	그 파일의 결과를 반드시 클라이언트의 웹 브라우저 이용해서 테스트해야만 한다. 
	
	Web Server
	: http 프로토콜(전송규약)에 의해서 클라이언트의 요청을 받아 해결하는 서버 지칭하는 단어
	
		[예]
			아파치, 톰캣, 레진, IIS, 엔진X,
	
		JSP처럼 클라이언트의 요청에 따라 동적으로 응답 문서를 만들어주는
		어플리케이션을 실행할 수 있는 서버를 Web Application Server(WAS)라고 부른다.
		
		[참고]
			프로젝트 만들고
			서버 만든 후
			서버에 프로젝트를 추가한 후 확인해야할 내용
				server.xml
				web.xml
			을 필요한 내용으로 되어있는지 확인
		
		[참고]
			프로젝트에서 사용할 파일들은 인코딩을 UTF-8로 설정
			
======================

	<프로젝트 폴더 구조>
	
		src	: 프로젝트 기본 폴더.. 프로젝트 관련된 파일들은 이 폴더 하위에 들어가야 한다.						
		src/main : 실제 프로젝트 소스 코드들이 작성되는 곳
		src/main/java : 프로젝트의 자바 소스코드가 작성되는 곳
		src/main/java/webapp : 프로젝트의 뷰에 관련된 문서들이 저장되는 곳
								==> 실제 JSP, HTML, JS, CSS 등 응답에 필요한 문서 작성
								
								META-INF : 데이터베이스 설정에 관련된 문서 저장
								WEB-INF : 서버에 관련된 설정들 또는 자바 클래스가 뷰를 저장했다가 부르는 폴더
								=> 이 두 폴더는 클라이언트가 접근 불가
									서버만 접근 가능
		
		[참고]
			XML(Extensible Markup Language) 파일
			: 확장가능한 마크업 언어(태그로 이루어진 문서)
			
			태그 이용해서 파일 내용 구분하도록 만들어지는 문서
			그 중 하나가 html 문서
			
			이때 html의 경우 사용 태그가 이미 결정
			xml의 경우 사용자가 트개 종료 정한 후 문서 만든다
			
			이 문서는 주로 프로그램 환경 설정을 위한 목적으로 많이 사용

			프로그램 실행 전 먼저 xml 파일 읽은 후 그 파일의 태그 분석
				=> 이 작업이 '파싱'
				
			파싱된 결과를 프로그램에서 이용
			
			
			결국 어떤 프로그램의 실행 환경(시작 환경) 바꾸고 싶으ㅕㄴ
			xml 파일에 환경 설정하는 부분 찾아서 수정 
			
			
			파싱 위해서 xml 문서에 있는 태그의 의미를 알아야 한다
			xml 문서를 만드는 사람이 태그의 순서, 구조, 의미를 지정하는 문서가 따로 잇는데
			이 문서를 'DTD' 또는 '스키마 문서'라고 한다
